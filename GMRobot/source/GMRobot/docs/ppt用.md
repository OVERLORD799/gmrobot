### 第一部分：一句话定位（1 页）

> 在论文的五阶段 VLM 管线基础上，搭建了**三层递进叠加的实时安全推理系统**，跑通了从仿真到感知栈到 VLM 联调的完整链路。论文是概念设计，我们做的是工程落地 + 架构增强。

### 第二部分：架构总览 — 实际实现（2–3 页）

#### 三层递进叠加架构

| 层 | 延迟 | 频率 | 职责 | 状态 |
|:--|:----|:----|:----|:----:|
| **Layer 1** 规则安全层 | < 1 ms | 50 Hz | 距离/TTC 硬实时门控，永不离线 | ✅ |
| **Layer 2** 数据驱动层 | < 50 ms | 50 Hz | 随机森林分类器，精细决策边界 | ✅ |
| **Layer 3** VLM 推理增强层 | ~850 ms | ~1 Hz 异步 | 语义理解、自然语言解释、感知定位 | ✅ MVP |
| **Motion Replan** | 按需 | 按需触发 | 主动式轨迹规避（非纯 STOP） | ✅ |

**对照论文**：论文的五阶段 VLM 管线 → 全部归入本系统的 Layer 3 + Motion Replan。额外增加的 Layer 1/2 作为实时安全底线。

#### 安全门控决策流

```
20 Hz 实时循环（永久在线）
┌──────────────────────────────────────┐
│ Layer 1: 规则安全层  (< 1 ms)         │ ← 始终启用
│  输出: ALLOW / SLOW_DOWN / STOP      │
└──────────────┬───────────────────────┘
               ↓
┌──────────────────────────────────────┐
│ Layer 2: 数据驱动层  (< 50 ms)        │ ← 在线融合
│  Tier 融合: ML 只降级不升级            │
└──────────────┬───────────────────────┘
               ↓
┌──────────────────────────────────────┐
│ Layer 3: VLM 推理增强层  (~1 Hz)      │ ← 异步，不阻塞门控
│  输出: 风险分类 + 自然语言解释         │
└──────────────┬───────────────────────┘
               ↓
┌──────────────────────────────────────┐
│ Motion Replan（Phase 4）              │ ← 主动式规避
│  3 种 held-aware 绕行策略             │
└──────────────────────────────────────┘
```

### 第三部分：关键工程问题与解决（4–5 页）

> 叙事逻辑：**遇到什么问题 → 怎么解决的 → 量化效果**

#### 3.1 纯 VLM 太慢 → 三层递进架构

| 要素 | 内容 |
|:-----|:-----|
| **问题** | VLM 推理需数百 ms~数秒，实时控制循环 50 ms/步，延迟不匹配 |
| **解决** | L1 规则（< 1 ms）+ L2 ML（< 50 ms）做实时底线；L3 VLM（~1 Hz）异步辅助，不阻塞门控；任一 VLM 宕机 → 安全不降级 |
| **量化** | 50 Hz 实时门控，论文目标 20 Hz（2.5×） |

**数据支撑**：架构总览 §5（能力边界表）、§2（三层架构）；控制频率见项目进展 §7.4（`control_dt=0.02`，sim 200 Hz × decimation 4）。

---

#### 3.2 规则太死 → false_stop 38.8% → ML 融合降误停

| 要素 | 内容 |
|:-----|:-----|
| **问题** | 在 shoulder_pass 和 stress preset 上，纯规则门控 `g_rule` false_stop 高达 38.8%，机器人频繁无故停住 |
| **解决** | 引入 Layer 2 随机森林分类器（GT 标签训练，`g_ml` false_stop = 0%）+ **Tier 融合策略**（ML 只降级不升级：g_rule=ALLOW 时恒 ALLOW；g_rule=STOP 时若 ML 判断 ALLOW 且置信度足够 → 降级） |
| **量化** | shoulder_pass: 38.8% → **0.0%**；stress: 38.8% → **0.03%** |

**数据支撑**：

| preset | run_id | g_rule false_stop | would_fuse false_stop |
|:-------|:-------|------------------:|----------------------:|
| `ivj_static_shoulder_pass` | `20260619_222109` | 38.8% | **0.0%** |
| `safety_layer1_stress` | `20260617_193713` | 38.8% | **0.03%** |

模型：`20260619_220454`（layer2_v2 30 维特征），来源：项目进展 §3.6。

#### 3.3 全包络门控反致任务卡死 → tier0_allow

| 要素 | 内容 |
|:-----|:-----|
| **问题** | 引入全包络门控（gripper 指尖球 + 夹持物盒 + 臂段 link 等 15+ 额外空间原语）后，`dist_min` 总是进入 Tier0 硬区（0.13 m）→ STOP 冻结 `time_step` → baseline `task_ts` 仅 **~286**，任务几乎无法推进 |
| **解决** | `tier0_allow`：包络距离 `dist_min` 在硬区但末端执行器 `dist_ee_human` 仍远（> safe_dist_warn）→ 判定为人手与夹持物盒接近，机械臂本体尚远 → ALLOW 放行 |
| **量化** | task_ts: 286 → **1777**（tier0_allow）→ **2015**（+ 人手轨迹 v2 + 视觉微调） |

**数据支撑**：项目进展 §3.7（S3 速度策略复评，baseline≈286 = tier0 锁死，tier0_allow → ≈1777）；§3.8.1（S1 REF_TIME_STEP=2015）。

---

#### 3.4 人手扫掠撞落零件 → 3+1 层漏件防御

| 要素 | 内容 |
|:-----|:-----|
| **问题** | kinematic `human_hand` 球体扫过夹持零件时将零件撞落，gripper boost 不足以避免 |
| **解决** | ① 碰撞冷却（5 步内不重复触发重抓）② `dist_min_held` 物理检测（< 0.06 m 立即回退至 `move_above`）③ VLM 视觉监督（连续 3 帧确认丢失 → 触发重抓）④ 姿态稳定保持（回退后 60 步朝向收敛） |
| **量化** | 3 策略 held-aware 绕行全部通过：`raise_then_lateral` ✅ / `lateral_first` ✅ / `retreat_then_arc` ✅（Isaac run `20260622_190148`） |

**数据支撑**：项目进展 §3.8.3、§3.9 W1/W14。

---

#### 3.5 VLM 不认识仿真场景 → prompt 工程 + 阈值门控

| 要素 | 内容 |
|:-----|:-----|
| **问题** | Qwen2.5-VL-7B 是通用 VLM，未见过 Isaac Sim 渲染的俯视机器人工作台图像，不知道"夹爪持件搬运"是安全的，倾向于将正常操作判为风险 |
| **解决** | 定制 `SAFETY_SYSTEM_PROMPT` 明确定义正常/异常状态 + `_build_vlm_context()` 传入当前任务阶段（approach/grasp/transit/place）+ `risk_confidence ≥ 0.85` + `risk_type == "dynamic"` 双重门控防止误触发 |
| **量化** | Isaac 500 步联调通过（run `20260622_130525`），`vlm_*` 列 500/500 非空 |

**数据支撑**：项目进展 §8.1、§8.5。

## PPT 制作要点

1. **数字大字展示**：38.8% → 0%、50 Hz、131 tests——数字比文字有说服力
2. **每页一个问题-解决-量化**三段式，是天然的 slide 素材
3. **放仿真截图**：Isaac Lab 环境图、VLM 推理帧、GUI 目视截图
4. **避免大段文字**：听众知道论文，不需要复述背景
5. **数据可追溯**：关键数字旁标注 run ID，应对追问
